### Добавление функционала получения погоды в админ панель Django

**Цель задания:**
Реализовать функционал, при котором по нажатию кнопки в админ панели Django происходит запрос к API OpenWeatherMap, получение данных о погоде для Москвы (координаты: lat=55.75, lon=37.61) и сохранение этих данных в новую модель Weather. Процесс должен быть реализован в виде отдельного модуля, который можно вызывать из админки.

**Требования и шаги реализации:**

1. **Настройка API ключа:**
   - В файле `simple/settings/base.py` добавить переменную `OPENWEATHER_API_KEY` с вашим API ключом от OpenWeatherMap (получить на сайте openweathermap.org).
   - Пример: `OPENWEATHER_API_KEY = 'your_api_key_here'`

2. **Создание модели Weather:**
   - В файле `simple/models/weather.py` создать Django модель `Weather` на основе структуры JSON ответа от API.
   - Основные поля модели (использовать подходящие типы Django: FloatField, CharField, IntegerField, DateTimeField и т.д.):
   - Зарегистрировать модель в `simple/models/__init__.py` и создать миграцию (`manage.py makemigrations` и `manage.py migrate`).

3. **Реализация процесса получения погоды:**
   - В файле `simple/processes/get_weather.py` создать функцию или класс для выполнения запроса.
   - Логика:
     - Использовать библиотеку `requests` для GET запроса к URL: `https://api.openweathermap.org/data/2.5/weather?lat=55.75&lon=37.61&appid={API_KEY}` (вставить API_KEY из settings).
     - Обработать JSON ответ (пример структуры ниже).
     - Создать в модели Weather с данными из ответа.
     - Обработать ошибки: если запрос не удался (например, 401 Unauthorized — неверный ключ, или 404 — город не найден), логировать ошибку и не сохранять данные.
     - Логировать процесс (успех/ошибку) в лог файл (использовать logging из Django).
   - Функция должна быть импортируемой и вызываемой из админки.

4. **Добавление кнопки в админ панель:**
   - В файле `simple/admin/admin.py` (или соответствующем для модели) добавить admin action или custom кнопку для модели.
   - Кнопка должна вызывать процесс из `get_weather.py` и отображать сообщение об успехе/ошибке (использовать `messages` из Django).
   - После нажатия кнопки перенаправлять обратно в админку с уведомлением.

5. **Тестирование:**
   - Проверить работу: зайти в админ панель, нажать кнопку, убедиться, что данные сохранились в БД (проверить через админку).
   - Обработать edge cases: отсутствие интернета, неверный API ключ, невалидный ответ от API.
